# 프로젝트 아이디어 생성기

새로운 프로젝트 아이디어 3개를 생성하는 스킬.

## 실행 절차

### 1단계: 사용자 파악

아이디어를 생성하기 전에, 대화 맥락에서 다음 정보를 파악한다:

- **기술 스택**: 사용자가 주로 사용하는 언어/프레임워크 (알 수 없으면 질문)
- **관심 분야**: 웹, 모바일, AI, 데이터, 게임, 자동화 등 (알 수 없으면 질문)
- **난이도 선호**: 초급/중급/고급 (알 수 없으면 중급으로 가정)

정보가 충분하지 않으면 간단히 질문한 뒤 진행한다.

### 2단계: 아이디어 생성

반드시 **정확히 3개**의 프로젝트 아이디어를 생성한다. 각 아이디어는 다음 형식을 따른다:

```
### [번호]. [프로젝트 이름]

> [한 줄 설명]

- **기술 스택**: [주요 기술 나열]
- **난이도**: [초급 / 중급 / 고급]
- **핵심 기능**:
  1. [기능 1]
  2. [기능 2]
  3. [기능 3]
```

### 3단계: 생성 원칙

아이디어 선정 시 다음 원칙을 따른다:

- 3개의 아이디어는 서로 **다른 유형**으로 구성 (예: 웹앱, CLI 도구, API 서비스 등 혼합)
- 실제로 **만들 수 있는** 현실적인 범위의 프로젝트 제안
- 가능하면 **포트폴리오에 도움이 되는** 실용적인 프로젝트 위주로 추천
- 최신 트렌드와 실제 수요가 있는 주제 우선 고려
- 사용자의 기술 수준에 맞는 적절한 도전 수준 설정

### 4단계: 마무리

3개의 아이디어를 제시한 후, 사용자에게 다음을 안내한다:

- 마음에 드는 아이디어가 있으면 바로 구현을 시작할 수 있다는 점
- 다른 방향의 아이디어를 원하면 추가 요청이 가능하다는 점
